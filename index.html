What about automation

  • Since selenium can do both testing and automation, I saw no reason not to implement a simple automation system.
  • All you have to do is return values from your test functions
  • Like so: module.exports = { 'Simple Test': wind => wind.getTitle() }

Keep it stupid simple

  • With Fastest.io you can test all DOM elements of a webpage
  • It is always asynchronous! You don't have to worry about async / await words
  • You can have more than one test in a single in a single javascript file and by doing it so you can have more than one test scenario

Test in every web driver available

  • With the directive --browser jsdom you can run tests in the jsdom enviorment
  • It is the most cheapest memory consuming web driver available
  • But it will support Google Chrome webdriver and others..

Is it free?

  • It is free for now
  • But if it reaches a large amount of active users it might turn into a pay-per-use service
